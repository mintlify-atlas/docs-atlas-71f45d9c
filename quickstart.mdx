---
title: "Get started with PROJEIC Backend"
description: "Learn how to authenticate with Google OAuth, obtain a JWT token, and make your first GraphQL query to the PROJEIC API in four steps."
---

PROJEIC Backend uses Google OAuth 2.0 for authentication. Every request to the GraphQL API must include a JWT bearer token obtained after completing the OAuth flow. This guide walks you through authentication and your first queries.

<Note>
  You need a running PROJEIC Backend instance before following this guide. See [Deploy PROJEIC Backend with Docker](/deployment) to get one running locally or in your environment.
</Note>

<Steps>
  <Step title="Authenticate with Google OAuth">
    Redirect the user's browser to the Google OAuth endpoint. PROJEIC will handle the OAuth handshake with Google and redirect back to your frontend with a JWT token.

    ```bash
    GET http://localhost:4000/projeic/api/auth/google
    ```

    After a successful login, the server redirects to your configured `FRONTEND_URL` with the JWT token. Store this token — you will include it in every subsequent API request as an `Authorization: Bearer` header.
  </Step>

  <Step title="Make a GraphQL request">
    All API operations go through a single GraphQL endpoint. Send a `POST` request with your query or mutation in the request body and include your JWT token in the `Authorization` header.

    ```bash
    curl --request POST \
      --url http://localhost:4000/projeic/api/graphql \
      --header 'Authorization: Bearer YOUR_JWT_TOKEN' \
      --header 'Content-Type: application/json' \
      --data '{"query":"{ me { userId name email isAdmin } }"}'
    ```

    <Note>
      Replace `YOUR_JWT_TOKEN` with the token received from the OAuth redirect. Requests without a valid token will be rejected with an `Unauthorized` error.
    </Note>
  </Step>

  <Step title="Query your profile">
    Use the `me` query to verify your authentication and retrieve your user profile.

    ```graphql
    query {
      me {
        userId
        name
        email
        isAdmin
      }
    }
    ```

    A successful response confirms your token is valid and returns your account details, including whether your account has admin privileges.
  </Step>

  <Step title="List your projects">
    Use the `myProjects` query to fetch the projects you belong to. The response is paginated and includes a `total` count.

    ```graphql
    query {
      myProjects {
        items {
          id
          name
          status
          methodology
        }
        total
      }
    }
    ```

    Each project returns its current `status` (`STARTING`, `ACTIVE`, `ON_HOLD`, `COMPLETED`, or `CANCELLED`) and `methodology` (`KANBAN`, `SCRUM`, `SCRUMBAN`, or `NONE`).
  </Step>

  <Step title="Create a project">
    Use the `createProject` mutation to create a new internal project. Provide at minimum a `name`; all other fields are optional.

    ```graphql
    mutation {
      createProject(input: {
        name: "My Engineering Project"
        description: "A new internal project at EIC"
        methodology: SCRUM
        isPublic: false
        isInstitutional: false
      }) {
        id
        name
        status
        methodology
        createdAt
      }
    }
    ```

    The mutation returns the newly created project. The `status` defaults to `ACTIVE` and the `methodology` you specified is applied immediately.
  </Step>
</Steps>
