title: "PROJEIC GraphQL mutations reference"
description: "Complete reference for all PROJEIC GraphQL mutations — create and manage projects, tasks, sprints, boards, members, and notifications."
---

PROJEIC mutations allow you to create, update, and delete resources across every domain in the platform. All mutations are sent as `POST` requests to `/projeic/api/graphql` with an `Authorization: Bearer <jwt>` header. The sections below document every available mutation, its input fields, return type, and a usage example. See the [API overview](/api/overview) for request format details.

---

## Projects

### `createProject`

Creates a new project and adds the authenticated user as its leader.

```graphql
mutation CreateProject($input: CreateProjectInput!) {
  createProject(input: $input) {
    id
    name
    status
    methodology
    color
    isPublic
    createdAt
  }
}
```

**Input: `CreateProjectInput!`**

<ParamField body="name" type="string" required>
  Project display name (3–100 characters).
</ParamField>
<ParamField body="description" type="string">
  Optional project description (max 500 characters).
</ParamField>
<ParamField body="color" type="string" default="#3B82F6">
  Hex color used for UI theming (e.g. `#3B82F6`).
</ParamField>
<ParamField body="methodology" type="ProjectMethodology" default="KANBAN">
  Development methodology: `KANBAN`, `SCRUM`, `SCRUMBAN`, or `NONE`.
</ParamField>
<ParamField body="mode" type="ProjectMode">
  Management mode: `CLASSIC` or `HYBRID`.
</ParamField>
<ParamField body="status" type="ProjectStatus" default="ACTIVE">
  Initial lifecycle status: `ACTIVE`, `STARTING`, `ON_HOLD`, `COMPLETED`, or `CANCELLED`.
</ParamField>
<ParamField body="isPublic" type="boolean" default="false">
  Make the project visible to all users.
</ParamField>
<ParamField body="isInstitutional" type="boolean">
  Mark the project as institutional.
</ParamField>
<ParamField body="professorId" type="string">
  Associate the project with a professor by user ID.
</ParamField>
<ParamField body="subjectId" type="string">
  Associate the project with a subject by ID.
</ParamField>

**Returns:** `ProjectEntity!`

### `archiveProject`

Soft-deletes a project by setting its `isArchived` flag to `true`. The project remains in the database and can be included in queries by passing `includeArchived: true`.

```graphql
mutation ArchiveProject($id: ID!) {
  archiveProject(id: $id) {
    id
    name
    isArchived
  }
}
```

<ParamField body="id" type="ID" required>
  ID of the project to archive.
</ParamField>

**Returns:** `ProjectEntity!`

---

### `deleteProject`

Permanently deletes a project and all its associated data via cascading delete. This action is irreversible.

<Warning>
  `deleteProject` permanently removes the project and all related tasks, sprints, boards, members, and activity logs. There is no recovery path.
</Warning>

```graphql
mutation DeleteProject($id: ID!) {
  deleteProject(id: $id) {
    id
    name
  }
}
```

<ParamField body="id" type="ID" required>
  ID of the project to permanently delete.
</ParamField>

**Returns:** `ProjectEntity!`

```graphql

    role
    status
    joinedAt

      id
      name
      avatarUrl
    }
  }
}
```

**Input: `AddProjectMemberInput!`**

<ParamField body="email" type="string" required>
  Email address of the user to invite.
</ParamField>
<ParamField body="projectId" type="ID" required>

</ParamField>
<ParamField body="role" type="ProjectRole" required>
  Role to assign: `LEADER`, `STUDENT`, `SUPERVISOR`, or `EXTERNAL`.

### `updateResultStatus`

Updates the status of an expected result and optionally attaches evidence.

```graphql
mutation UpdateResultStatus($input: UpdateResultStatusInput!) {
  updateResultStatus(input: $input) {
    id
    title
    status
    progress
    updatedAt
    evidences {
      id
      type
      url
      fileKey
    }
    history {
      id
      previousStatus
      newStatus
      reason
      createdAt
    }
  }
}
```

**Input: `UpdateResultStatusInput!`**

<ParamField body="resultId" type="string" required>

</ParamField>

</ParamField>

  Reason for the status change, stored in the status history log.
</ParamField>
<ParamField body="evidenceType" type="string">

</ParamField>
<ParamField body="evidenceUrl" type="string">
  Public URL pointing to the evidence.
</ParamField>
<ParamField body="evidenceFileKey" type="string">
  S3 file key for evidence uploaded via `getUploadPresignedUrl`.
</ParamField>

**Returns:** `ExpectedResult!`

## Files

### `getUploadPresignedUrl`

Generates a pre-signed S3 upload URL for direct browser-to-storage uploads. Use the returned `uploadUrl` to `PUT` the file, then pass the `fileKey` to `updateResultStatus` as evidence.

```graphql
mutation GetUploadPresignedUrl($contentType: String!, $fileName: String!) {
  getUploadPresignedUrl(contentType: $contentType, fileName: $fileName) {
    uploadUrl
    fileKey
  }
}
```

<ParamField body="contentType" type="string" required>
  MIME type of the file (e.g. `application/pdf`, `image/png`).
</ParamField>
<ParamField body="fileName" type="string" required>
  Original file name, used to derive the storage key.
</ParamField>

**Returns:** `PresignedUrlResponse!`

<ResponseField name="uploadUrl" type="string" required>
  Pre-signed S3 URL. Send a `PUT` request with the file body directly to this URL.
</ResponseField>
<ResponseField name="fileKey" type="string" required>
  The S3 object key to reference the uploaded file in subsequent API calls.
</ResponseField>
